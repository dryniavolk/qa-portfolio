## 🧪 Проект 1: Автоматизация тестирования Aviasales (API + UI)

**Задача:**  
Разработать фреймворк для автоматизации тестов поиска авиабилетов, поддерживающий запуск только UI, только API или всех тестов, с генерацией Allure-отчётов.

**Что сделано:**
- Реализована **Page Object Model** для главной страницы и страницы результатов.
- Написан **API-клиент** для поиска билетов (туда-обратно, в один конец, с младенцем), управляющий `search_id` и `X-Request-Id`.
- Создан **CookieManager**, который кэширует cookies в файл (TTL 30 минут) и обновляет их через headless Selenium.
- HTTP-клиент автоматически подставляет заголовки (`Origin`, `Referer`, `User-Agent`, `Cookie`).
- Написаны **pytest-тесты** с маркерами `ui` и `api`, параметризацией и Allure-шагами.
- Настроена фикстура `driver` с маскировкой автоматизации (отключение `navigator.webdriver`).
- Составлена **Postman-коллекция** из 10 тестов (5 API + 5 UI) с динамическими переменными.

**Технологии:**  
Python 3.14, pytest, Selenium, Requests, Allure, Postman, webdriver-manager, Git.

**Результаты:**  
✅ 10 автотестов стабильно проходят многократные прогоны.  
✅ Проект поддерживает три режима запуска, данные вынесены в конфиги.  
✅ Allure-отчёты содержат шаги, скриншоты (UI) и логирование.  
✅ Код соответствует PEP8, использованы аннотации типов.

🔗 **Ссылка на репозиторий:** https://github.com/dryniavolk/pytest_ua_api_template.git

## 📅 Проект 2: Тестирование функционала «Личные события» в расписании онлайн-школы Skyeng

**Задача:**  
Провести полный цикл тестирования нового функционала (добавление, редактирование, удаление личных событий), включая API-тестирование, регресс, оформление отчёта.

**Что сделано:**
- Проанализировал требования, выявил неоднозначности (создание событий в прошлом, максимальная длина названия и др.).
- Составил **тест-план**, **декомпозицию** (Miro), **smoke-тест**, **функциональные чек-листы**, **регрессионный чек-лист**, **приёмочные тест-кейсы**.
- Вручную протестировал сценарии, нашёл **5 багов** (critical: удаление урока с портала учителя, создание события в прошлом; major: ввод только знаков препинания; minor: ограничение 40 символов, цвет события).
- Провёл регрессионное тестирование — обнаружил критическую поломку старого функционала (добавление/удаление/перенос урока).
- Для API-тестов в Postman настроил:
  - переменные окружения (`baseUrl`, `token`, `eventIdOne`, `startAt`, `endAt`);
  - pre-request скрипты (генерация дат, проверка длины названия >40 символов);
  - тесты на статус-коды и сохранение ID событий.
- Оформил **баг-репорты** в трекере «Ёжка», подготовил итоговый **отчёт** с выводами и рекомендациями.

**Технологии:**  
DevTools, Postman, Test IT, Miro, баг-трекинг «Ёжка».

**Результаты:**  
✅ Выявлено 5 дефектов нового функционала и 1 критическая поломка старого.  
✅ API-тесты автоматизированы в Postman (коллекция с переменными и скриптами).  
✅ Дана рекомендация: выпуск возможен только после исправления критических и высокоприоритетных багов.
